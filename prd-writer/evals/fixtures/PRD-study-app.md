# study-app PRD

## 概述
- 产品定位: 一款帮助用户养成每日学习习惯的打卡小程序
- 目标用户: 自学者、备考人群、需要监督养成学习习惯的人

## 功能详情

### F001 - 用户注册登录

**功能描述**
用户通过手机号 + 验证码完成注册和登录,首次进入自动创建账号。

**使用场景**
新用户首次打开 app 需要注册,老用户每次启动 app 需要登录(或保持登录态)。

**交互流程**
1. 用户打开 app,进入登录页
2. 用户输入手机号(11 位,以 1 开头) → 系统校验格式
3. 用户点击"获取验证码" → 系统下发短信 → 按钮置灰倒计时 60s
4. 用户输入 6 位数字验证码 → 点击"登录" → 系统校验
5. 校验通过 → 进入首页;首次登录自动创建账号

**异常处理**
- 手机号格式错误 → 输入框下方红字提示
- 验证码错误 → toast 提示"验证码错误,请重新输入"
- 验证码过期(超过 5 分钟) → toast 提示"验证码已过期,请重新获取"
- 60s 内重复点击发送 → 按钮置灰,显示剩余倒计时

**边界与约束**
- 验证码有效期 5 分钟
- 同一手机号 1 小时内最多发送 5 次验证码
- 用户昵称在首次登录时自动生成,格式为"学习者 + 4 位随机数字"
- 本功能**不**包含: 第三方账号登录(微信/Apple ID)、密码登录、邮箱登录

### F002 - 每日打卡

**功能描述**
用户每天可以发布一条打卡内容(必填文字 + 可选图片),用于记录今天的学习。

**使用场景**
用户完成当天学习后,在首页点击"打卡"按钮记录学习内容。

**交互流程**
1. 用户在首页点击"打卡"按钮 → 进入打卡编辑页
2. 用户输入打卡文字(必填,1-500 字)
3. 用户可选上传 1-3 张图片(可选)
4. 用户点击"发布" → 系统校验 → 提交打卡 → 返回首页,显示"今日已打卡"

**异常处理**
- 文字为空时点击发布 → toast 提示"请输入打卡内容"
- 文字超过 500 字 → 输入框下方显示字数超限,无法发布
- 当日已打卡再次进入打卡页 → 显示今日打卡内容,可编辑/删除
- 图片上传失败 → 该图片显示"重试"按钮,不影响发布(可不带该图)

**边界与约束**
- 每个用户每天最多打卡 1 次(0:00 - 24:00 为一天)
- 单次打卡文字 1-500 字
- 单次打卡最多 3 张图片,单张图片不超过 5MB
- 当日打卡可在当天内修改或删除,次日起不可修改
- 本功能**不**包含: 打卡内容的点赞/评论/分享、定时提醒打卡
