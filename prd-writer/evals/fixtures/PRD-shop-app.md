## 功能详情

### F001 - 用户注册登录

**功能描述**
用户通过手机号 + 密码完成注册和登录。注册时需要设置用户名和密码。

**使用场景**
新用户在浏览或下单前需要注册账号;老用户每次启动 app 需要登录(或保持登录态)。未登录用户可以浏览商品,但不能加购或下单。

**交互流程**
1. 用户进入登录/注册页
2. 注册: 输入手机号 → 获取验证码 → 设置用户名(4-16 位字母数字) → 设置密码 → 提交
3. 登录: 输入手机号 + 密码 → 提交 → 进入首页

**异常处理**
- 手机号已注册 → toast 提示"该手机号已注册,请直接登录"
- 用户名格式不符(非 4-16 位字母数字) → 输入框下方红字提示
- 用户名已被占用 → 输入框下方红字提示
- 密码错误 3 次 → 锁定该账号 15 分钟

**边界与约束**
- 用户名必须 4-16 位,只能包含字母和数字
- 密码必须 8-20 位,包含字母和数字
- 同一手机号只能注册一个账号
- 本功能**不**包含: 第三方账号登录、找回密码(下一版本)

### F002 - 商品浏览

**功能描述**
用户可以在首页和分类页浏览商品,查看商品详情。

**使用场景**
用户进入 app 后,在首页看到推荐商品流,可以点击进入分类筛选,或点击具体商品看详情。

**交互流程**
1. 用户在首页查看推荐商品瀑布流
2. 用户可点击顶部分类 tab,切换到不同分类的商品列表
3. 用户点击商品卡片 → 进入商品详情页 → 查看图片、价格、描述、规格

**边界与约束**
- 未登录用户可以浏览所有商品和商品详情
- 商品详情页加购按钮在未登录时点击会引导去登录
- 本功能**不**包含: 商品搜索(下一版本)、商品收藏

### F003 - 加入购物车

**功能描述**
登录用户可以将商品加入购物车,在购物车页面查看、修改数量、删除商品。

**使用场景**
用户在商品详情页选好规格后加入购物车,稍后在购物车页统一结算。

**交互流程**
1. 用户在商品详情页选择规格(尺码、颜色) → 点击"加入购物车"
2. 系统提示"已加入购物车",购物车 icon 数量 +1
3. 用户进入购物车页 → 查看商品列表,可修改单品数量、删除商品

**边界与约束**
- 购物车单品数量上限 99 件
- 购物车最多 50 个不同 SKU
- 未登录用户点击加入购物车 → 引导去登录,登录后自动加入
- 本功能**不**包含: 跨设备同步购物车(本地存储)
